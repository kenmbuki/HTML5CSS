Chapter 8 - Padding, Borders, and Margins

borders can set an element apart from others, accentuate its appearance, mark certain kinds of data as having been changed, or any number of other things. Between the content of an element and its border, we find the padding of an element, and beyond the border, the margins. These properties affect how the entire document is laid out, of course, but more importantly, they very deeply affect the appearance of a given element.

the element box, which describes the amount of space that an element occupies in the layout of the document.

There are three ways to generate additional space around elements: you can add padding, margins, or a combination of padding and margins.

easy way to remember the order in which sides must be declared is to keep in mind that getting the sides in the correct order helps you avoid “TRouBLe” —that is, TRBL, for “Top Right Bottom Left.”

There are four ways to assign width to a border: you can give it a length value such as 4px or 0.1em or use one of three keywords. These keywords are thin, medium (the default value), and thick.

Summary

The ability to apply margins, borders, and padding to any element is one of the things that sets CSS so far above traditional web markup. In the past, enclosing a heading in a colored, bordered box meant wrapping the heading in a table, which is a really bloated and awful way to create such a simple effect. It is this sort of power that makes CSS so compelling.
